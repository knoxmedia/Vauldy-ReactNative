# Vauldy 移动终端客户端

> 需求规格说明书（SRS）

---

## 1. 项目概述

### 1.1 项目背景

Vauldy 是一款自托管的家庭数字保险箱，以 Go 后端 + React Web 前端形式提供影视、音乐、照片、文档等媒体的安全存储与在线浏览能力。随着用户在移动场景下的使用需求增长，需要为 Android 与 iOS 平台提供原生体验一致的移动终端客户端，使用户能够在手机和平板上随时随地访问家中 Vauldy 媒体服务器。

### 1.2 项目目标

基于 **React Native** 跨端框架，开发 Vauldy 移动终端客户端（以下简称「移动客户端」），实现：

- 对接现有 **Vauldy Media Server** REST API（`/api/v1`），无需修改后端核心协议
- 一套代码同时适配 **Android** 与 **iOS**
- 支持媒体库浏览及 **视频、音频、图片、文档** 在线预览/播放
- 提供与 Web 端消费者功能对等的日常使用体验（浏览、搜索、收藏、播放列表、续播等）
- 独立代码仓库维护（`knoxmedia/Vauldy-ReactNative`），通过 Git Submodule 嵌入主项目便于联调

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| Vauldy Media Server | 自托管媒体服务端，默认监听 `8200` 端口，提供 API 与流媒体服务 |
| 移动客户端 | 本项目的 React Native 应用，运行于 Android / iOS |
| 媒体库（Library） | 服务端按类型划分的媒体集合，类型包括 movie、tv、anime、music、photo、document 等 |
| 播放计划（Playback Plan） | `GET /media/:id/hls` 返回的 JSON，描述直链、HLS、DRM、JIT 等播放方式 |
| access_token | JWT 令牌；流媒体资源 URL 可通过查询参数 `?access_token=` 鉴权 |
| JIT 转码 | 按需实时转码生成 HLS 流，无需预先打包 |
| DRM | 内容加密方案，包括 Widevine（Android）、FairPlay（iOS）、PowerDRM、HLS AES-128 |

### 1.4 项目范围

**包含：**

- 终端用户（admin / user 角色）的日常消费功能
- 与服务端已有 API 的完整对接
- Android / iOS 双平台发布

**不包含（首期）：**

- 管理后台功能（库管理、扫描、转码任务、用户管理、系统选项等 `adm` 路由）
- 本地上传媒体文件
- 离线缓存完整媒体文件（可作为后续版本规划）
- 演职人员（Person）模块（待后端 API 路由正式注册后再纳入）

| 角色 | 描述 | 移动端权限 |
|------|------|------------|
| 管理员（admin） | 服务端管理员账号 | 与 Web 端一致：可浏览、播放所有已授权库；**不**提供管理控制台 |

## 4. 功能需求

### 4.1 服务器连接与认证

#### 4.1.1 服务器配置

**描述**：首次启动或设置页配置 Vauldy 服务器地址。

| 字段 | 必填 | 说明 |
|------|------|------|
| 服务器地址 | 是 | 如 `https://192.168.1.100:8200`，支持 HTTP/HTTPS |
| 连接测试 | — | 调用 `GET /health` 验证可达性 |

**验收标准**：
- 地址持久化存储，支持修改与清除
- 连接失败时给出明确错误提示（超时、证书、不可达）
- 支持保存多个服务器配置（可选，二期）

#### 4.1.2 用户登录

**描述**：使用用户名密码获取 JWT 会话。

| API | 方法 | 路径 |
|-----|------|------|
| 登录 | POST | `/api/v1/user/login` |
| 用户信息 | GET | `/api/v1/user/info` |
| 登出 | POST | `/api/v1/user/logout` |
| 品牌信息 | GET | `/api/v1/branding` |

**基本流程**：
1. 用户输入用户名、密码
2. 客户端调用登录接口，获取 `{ token, expires_in_hours }`
3. Token 加密存入本地安全存储
4. 跳转首页，后台拉取 `user/info` 填充角色与权限
5. 后续 JSON 请求携带 `Authorization: Bearer <token>`

**验收标准**：
- 登录成功进入首页；失败显示服务端错误信息
- Token 过期或 401 自动跳转登录页并清除本地会话
- 展示服务端 `branding.app_name` 作为应用标题（如有）

#### 4.1.3 权限与家长控制

**描述**：遵循服务端 `user/info` 返回的权限字段。

| 字段 | 行为 |
|------|------|
| `can_play` | 为 false 时禁止进入播放器 |
| `can_download` | 为 false 时隐藏下载按钮 |
| `library_scope: selected` | 仅展示已授权媒体库 |
| `parental_enabled` | 触发分级/时段限制时弹出 PIN 解锁 |

**PIN 解锁 API**：`POST /api/v1/user/parental/unlock` → `{ unlock_token, expires_in_minutes: 20 }`

### 4.6 视频播放

#### 4.6.1 播放流程

```
用户点击播放
    │
    ▼
GET /api/v1/media/:id/hls  ──► 播放计划 JSON
    │
    ├── mode: direct      ──► GET /media/:id/play（渐进式 MP4）
    ├── mode: hls         ──► HLS master URL
    ├── mode: hls_drm     ──► HLS + Widevine/FairPlay
    ├── mode: hls_powerdrm──► HLS + PowerDRM key
    ├── mode: hls_aes_128 ──► HLS + AES-128 key
    └── JIT session       ──► /jit/session/:id/master.m3u8
    │
    ▼
原生播放器加载（所有资源 URL 附加 ?access_token=）
    │
    ▼
POST /media/:id/playback/start（开始）
POST /media/:id/progress（定时上报进度）
POST /media/:id/playback/end（结束）
```

#### 4.6.2 播放器功能

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 播放/暂停 | P0 | 基础控制 |
| 进度条拖动 | P0 | Seek |
| 全屏/横屏 | P0 | 自动旋转 |
| 音量/亮度手势 | P1 | 行业惯例 |
| 倍速播放 | P1 | 0.5x ~ 2.0x |
| 字幕切换 | P1 | 加载 `/subtitles/:sid/vtt` |
| 多音轨切换 | P2 | HLS `/atrack/:stream/` |
| 预览缩略图 | P2 | sprite.jpg + thumbs.vtt |
| 画中画 (PiP) | P2 | iOS/Android 系统 PiP |
| DRM 加密内容 | P0 | 按库配置自动选择 Widevine/FairPlay/PowerDRM |

#### 4.6.3 DRM 平台适配

| DRM 类型 | Android | iOS |
|----------|---------|-----|
| Widevine | ExoPlayer + `/drm/widevine/license` | — |
| FairPlay | — | AVPlayer + `/drm/fairplay/cert` + `/drm/fairplay/license` |
| PowerDRM | 自定义 key handler + `/drm/powerdrm/key` | 同左或 WebView 降级 |
| HLS AES-128 | 标准 key URL `/drm/hls/aes128/key` | 同左 |

**验收标准**：
- 非 DRM 库视频正常 HLS/直链播放
- DRM 库在对应平台可完成 license 交换并播放
- 播放失败时展示可读错误（权限不足、转码中、格式不支持）
- 断点续播：启动时读取本地/服务端进度，seek 到上次位置

#### 4.6.4 JIT 实时转码

- 播放计划返回 `session_id` 时，使用 `/api/v1/jit/session/:sessionID/master.m3u8`
- 支持 `POST pause / resume / seek / end` 会话控制
- 首次缓冲显示 loading 状态，超时提示重试

## 5. 接口集成规范

### 5.1 认证方式

| 场景 | 认证方式 |
|------|----------|
| JSON API 请求 | Header: `Authorization: Bearer <JWT>` |
| 流媒体 / 图片 / HLS 分片 | Header 或 `?access_token=<JWT>` |
| 健康检查 | 无需认证 |

### 5.2 URL 构建规则

```typescript
// JSON API
const apiBase = `${serverUrl}/api/v1`;

// 流媒体资源（播放器无法设置 Header）
function assetUrl(path: string, token: string): string {
  const sep = path.includes('?') ? '&' : '?';
  return `${serverUrl}${path}${sep}access_token=${encodeURIComponent(token)}`;
}
```

### 5.3 错误处理

| HTTP 状态 | 客户端行为 |
|-----------|------------|
| 401 | 清除 Token，跳转登录页 |
| 403 | 展示权限不足提示（含家长控制 PIN 弹窗） |
| 404 | 媒体不存在或已删除 |
| 5xx | 服务端错误，支持重试 |

### 5.4 请求超时

- 普通 API：30 秒
- 流媒体/下载：120 秒
- 健康检查：5 秒

## 6. 非功能需求

### 6.1 性能

| 指标 | 目标 |
|------|------|
| 冷启动 | ≤ 3 秒（已登录状态） |
| 首页首屏 | ≤ 2 秒（局域网） |
| 缩略图加载 | 渐进显示，占位图兜底 |
| 视频起播 | ≤ 5 秒（预转码 HLS）；JIT 首片 ≤ 15 秒 |
| 内存 | 后台播放时 ≤ 150 MB 增量 |

### 6.2 兼容性

| 平台 | 最低版本 |
|------|----------|
| Android | 8.0（API 26） |
| iOS | 15.0 |

### 6.3 安全

- JWT 存储于系统安全存储（Keychain / EncryptedSharedPreferences）
- 支持 HTTPS；自签名证书需用户确认（开发/内网场景）
- 不在日志中输出 Token 或密码
- 可选：应用锁（生物识别/PIN，本地功能）

### 6.4 网络

- 优先局域网访问；支持远程访问（端口转发 / VPN）
- 弱网下降级：照片用 thumb 替代 medium，视频降低初始码率
- 无网络时展示离线提示（首期不做离线缓存）

### 6.5 可用性

- 支持横屏与竖屏（视频默认横屏，其余竖屏）
- 安全区域适配（刘海屏、底部 Home Indicator）
- 无障碍：基础屏幕阅读器支持（后期完善）
- 深色主题（默认），与 Vauldy Web 风格统一
